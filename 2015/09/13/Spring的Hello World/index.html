Spring的Hello World

By MiShuai

发表于 2015-09-13

Spring是一个从实际开发中抽取出来的框架，它完成了大量开发中的通用步骤，留给开发者的仅仅是与特定应用相关的部分，从而大大提高了企业开发的效率。

Spring为企业应用的开发提供了一个轻量级的解决方案，该方案包括：

1. 基于依赖注入的核心机制
2. 基于AOP的声明式事务
3. 与多种持久层技术的整合
4. 优秀的Web MVC框架

Spring致力于Java EE应用层各层解决方案，而不是仅仅专注于某一层的方案。可以说，Spring是企业应用开发的“一站式”选择，Spring贯穿表现层、业务层、持久层。

Spring具有如下优点：

1. 低侵入式设计，代码的污染极低。
2. Spring的DI容器降低了业务对象替换的复杂性，提高了组件之间的解耦。
3. Spring的AOP支持允许将一些通用任务如安全、事物、日志等进行集中式处理，从而提供了更好的复用。
4. Spring的ORM和DAO提供了与第三方持久层框架的良好整合，并简化了底层的数据库访问。
5. Spring的高度开放性，并不强制完全依赖于Spring，开发者可以自由选用Spring框架的部分或全部。

用Spring框架输出Hello Spring的小程序步骤如下：

  1. 上Spring官网下载Spring，我下载的是Spring4.2，创建一个Java工程，将所需jar包放入libs文件夹下。

  2. 创建一个HelloWorld类，添加一个私有属性name，并生成相应的get和set方法，并在set和get方法中用system.out.println()输出分别是什么方法，方便在控制台观察输出，再写一个hello()方法，输出属性名name，代码如下：

    public class HelloWorld {

    private String name;

    public HelloWorld() {
        System.out.println("构造方法");
    }

    public void setName(String name) {
        this.name = name;
        System.out.println("set方法");
    }

    public String getName() {
        System.out.println("get方法");
        return name;
    }

    public void hello() {
        System.out.println("Hello " + name);

    }

}

  3. 创建applicationContext.xml文件，在文件中配置bean，为HelloWorld类的name属性注入值，配置如下：

    <bean id = "hello"  class="com.spring.test.HelloWorld">
    <property name="name" value="Spring"></property>
</bean>

  4. 创建test类，并在类的main方法中使用Spring输出“Hello Spring”，代码如下：

    public class test {

    public static void main(String[] args) {

        // 创建 Spring 的 IOC 容器
        ApplicationContext ctx = new ClassPathXmlApplicationContext("applicationContext.xml");

        // 从 IOC 容器中获取 bean 的实例
        HelloWorld helloWorld = (HelloWorld) ctx.getBean("hello");

        helloWorld.hello();
    }

}

我们发现Spring容器在配置文件中注入赋值的时候调用了HelloWorld类的构造方法和name属性的set方法，说明Spring容器是通过类的构造方法创建对象，用set方法为类的属性赋值的，那么到底是在什么时候赋值的呢？我们不妨将main()方法中代码只留下第一行，而将后两行注释掉再看运行结果：

由上图可知Spring的IOC容器在启动的时候便会根据applicationContext.xml文件中配置的bean通过调用对应类的构造方法创建对象并调用对象的set方法为相应属性注入值，这就是Spring容器的核心：控制反转（依赖注入）。
