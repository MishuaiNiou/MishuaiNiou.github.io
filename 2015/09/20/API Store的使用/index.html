Api Store的使用

By MiShuai

发表于 2015-09-20

Android客户端与服务器端通信一个人来做是比较麻烦的，因为不仅要掌握客户端开发的知识，还要掌握服务器端的知识。
一般服务器传输数据到android客户端的思路是：首先写好服务器端，然后再将需要传输到android客户端的数据在servlet中封装成json的格式，最后在android客户端通过URLConnection或者HttpClient向服务器端请求数据，将服务器端封装好的json数据解析出来显示到UI界面中。

如果想快速的写出一款android客户端APP，百度的Api Store对于只会android客户端开发而对服务器端开发不熟的开发者来说真是一个福音。

Api Store是由百度出品，是一个连接服务商与开发者的第三方API分发平台。平台已汇集了国内外应用开发所需的Android/IOS API和SDK等800个服务，特有百度独家的检索抓取、语音图像、地图翻译等服务。首创即用API模式，使用百度账号即可调用平台内所有即用API，让开发工作如此简单。百度专区汇集了百度独家的文字识别（OCR）、百度地图、百度语音、百度翻译、百度云、百度站内搜索等服务；另免费提供天气、PM2.5、股票、IP地址、身份证等常用生活查询服务。

简而言之，有专门的服务商提供专门的服务，将他们想要提供的数据经过整理后封装成了json格式，并放在了Api Store上，开发者只要有百度账号，登录后就可通过服务商给出的URL地址和API文档获取免费的数据，android开发者就只需要按照文档解析json数据就OK了，省去了大量服务器端的工作，而这些复杂繁琐的工作服务商已经做好了，那么android开发者就可以快速开发APP了。

下面我举一个开发android新闻客户端的例子。

假如我想开发一款android新闻客户端的APP，那么，我原始的想法是：先写服务器端，用爬虫工具去爬取不同类别的新闻，爬取完后存储到数据库中，再将数据从数据库中取出在servlet中封装成json格式，android客户端再根据servlet的地址获取json格式数据并解析显示到界面上。这样的话，由于新闻每天都在变，所以数据库中的信息需要不停的更新，这是比较麻烦的；再就是爬虫框架的爬取策略，如何爬取才能爬取到最热门和最有价值的新闻，这又是一门学问了。所以，想做好是比较麻烦的。

现在使用Api Store之后，服务器端的新闻内容服务商已经帮忙筛选好了，并且也封装好了，每隔几个小时也会自动更新新闻，而android开发者就可以专注于移动端的开发了，根据服务商提供的文档直接解析封装好的json数据就OK了，多么方便啊！

接下来，说一下使用百度Api Store的大体步骤：

首先，你需要有百度的账号。登陆之后，搜索一下你需要的服务，例如我搜索新闻，会搜到很多种类的新闻，我选择了科技类新闻，进去后可以看到有提供好了的Api文档和各种编程语言示例。百度还会分给你一个apikey，这可能会需要手机认证之后才会看到。
